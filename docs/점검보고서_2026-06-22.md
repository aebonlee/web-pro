## 5. UI · UX 점검

- **상단 메뉴**: 3그룹(메인 · 학습 트랙 · 학습 도구) 구분선 + 트랙 색상 점. 트랙 라벨 정식 명칭 통일(웹 기초·React 프론트엔드·AI 웹 서비스·실전 배포·협업). 1101px↑ 한 줄, 1100px↓ 버거 메뉴 — 한 줄·미겹침 확인.
- **홈 트랙 카드 4종**: 크기 **314×383 동일** 정렬(높이 100%·제목 2줄 예약·keep-all 줄바꿈).
- **자료실**: 커리큘럼 챕터 + 코칭·가이드 섹션 동시 노출.
- 히어로 배경: JS 효과 도입 검토 후 최종 제거 → 원래 배경(silk + 그라데이션) 유지.

## 6. SEO · 배포 산출물

- **프리렌더**: 70개 라우트에 route별 title/description/OG 메타 정적 생성 — 크롤러 대응 양호.
- **OG 이미지**: `og-image-v2.png` 정상 생성(동적 카운트 반영, ~107KB).
- **커스텀 도메인**: `CNAME = bootcamp.dreamitbiz.com` 정상.
- **SPA fallback**: 배포 워크플로에서 `dist/404.html` 생성(딥링크/새로고침 404 방지) — 정상.
- 배포: `main` 푸시 → GitHub Actions(`npm ci → test → build → prerender → 404 fallback → Pages`) 자동 배포.

## 7. 개선 권장 사항 (결함 아님)

| 우선순위 | 항목 | 내용 |
| --- | --- | --- |
| 권장 | **sitemap.xml 부재** | 프리렌더 라우트 70개가 있으나 사이트맵이 없음. `public/sitemap.xml`(또는 빌드 생성)을 추가하면 검색 색인 효율↑. |
| 권장 | **robots.txt 부재** | `public/robots.txt`로 크롤링 정책 + sitemap 위치 명시 권장. |
| 선택 | 메인 번들 크기 | `index-*.js` ~520KB(gzip ~156KB). 이미 라우트/챕터 코드분할 적용. 추가 최적화는 선택. |
| 선택 | dev 문서 서식 | 개발일지(.md) markdownlint 서식 경고 — 기능 무관, 정리 선택. |

> 위 권장 4건은 모두 현재 동작에 영향이 없는 **개선 여지**입니다. 특히 sitemap.xml + robots.txt는 저비용·고효율 SEO 보강으로 우선 검토를 권합니다.
