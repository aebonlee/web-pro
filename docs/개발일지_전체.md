## 2. 개발 타임라인

### ① 초기 구축 — React·AI 학습사이트
- NOVAWORKS dc-runtime 아티팩트(블루 그라데이션·오렌지 포인트·Pretendard·플로팅 필 헤더)를 React 18 + Vite 5로 포팅.
- 교재 PDF 16종을 `pdftotext`로 추출 → 병렬 에이전트로 **16챕터 62강의** 커리큘럼 데이터화.
- 두 트랙(React 프론트엔드 / AI 웹 서비스), 페이지: Home/Curriculum/Track/Lesson/Login/MyPage/About/AuthCallback/404.
- **Supabase 연동**: 구글·카카오 OAuth + 섹션 단위 진도 저장(`webpro_` 접두사, RLS).
- OG 메타 + sharp OG 이미지, GitHub Actions → Pages 자동 배포.

### ② 리포 rename & 도메인
- `web-pro` → `bootcamp` 리네임, 커스텀 도메인 `bootcamp.dreamitbiz.com` 바인딩(Pages build_type=workflow).

### ③ 부트캠프 브랜딩 + 자료실·실습
- 브랜드 WEB PRO → **부트캠프(BOOTCAMP)** 전면 변경.
- 자료실(/resources), 챕터별 실습 예제 데이터.

### ④ 교재 전문 웹 콘텐츠화 (핵심)
- 16개 챕터 전체 본문을 **마크다운으로 변환(~21k줄)** — 설명·코드·실습 소스 전부 수록.
- 강의 상세 재구성: 좌측 목차(섹션 진도체크) + 우측 본문(react-markdown), 챕터별 lazy 청크.
- **PDF 비배포 원칙**: docs/pdf·text 제거, 웹 학습 콘텐츠로만 제공.

### ⑤ 코드 가독성 & 모바일
- 코드블록 **구문 강조(react-syntax-highlighter) + 주석 녹색(#3FB950) + 복사 버튼**.
- 전 챕터 코드에 상세 한국어 주석 보강.
- 모바일에서도 목차·진도 체크 노출(좌측 목차를 모바일 카드로).

### ⑥ 상단 메뉴 개선
- 영문 라벨 + 한글 병기(우측정렬·볼드·브랜드블루), 활성 표시를 밑줄로 통일, ABOUT 맨 앞.

### ⑦ 프로젝트 · 복습/퀴즈 메뉴
- /projects: 입문·중급·고급 프로젝트(기능·스택·관련강의 연결).
- /quiz: 핵심 용어(검색·트랙필터) + 퀴즈(챕터별/랜덤, 채점·해설·최고점 localStorage).

### ⑧ 배포·협업(ops) 트랙 신설
- 추가 PDF 2종(「AI·웹앱 배포 완전 정복」 12장, 「Git & GitHub 슬라이드」)을 분석·보강.
- 6개 챕터: Git·GitHub / AI챗봇 배포 / EC2·Docker·WinSCP / 프레임워크 AWS 배포 / 풀스택 통합 / 도커 MSA.

### ⑨ 프로젝트 상세 (실제 구현 + 동작 확인)
- /project/:id: **구현된 상태 확인 + 실제 구현 코드 + 관련 학습자료**.
- 프론트엔드 6종은 **실제 동작 라이브 데모**(데모 컴포넌트 = 표시 소스 일치, `?raw`).
- 백엔드·AI·배포 12종은 챕터 기반 실행 코드 + 예상 결과(JSON/터미널/설명).
- 프로젝트 레벨당 4 → 6개(총 18) 확장.

### ⑩ 웹 기초(web) 트랙 신설 — REACT 앞
- 5개 챕터: HTML 구조 / CSS 기초 / CSS 레이아웃 / JS 문법 / JS와 브라우저(DOM·이벤트·fetch).
- 메뉴·커리큘럼에서 REACT 앞에 배치, 바이올렛 컬러.

### ⑪ 점검·정비
- OG 한글 깨짐(우분투 CI 폰트 부재) → CI 재생성 제거 + 로컬 정상본 배포, 캐시 무력화 위해 `og-image-v2.png`로 파일명 버전.
- 메인 커리큘럼 프리뷰 4+2 → **3×3(9장)** 정렬.
- 전체 사이트 재점검(QA) + 평가보고서 작성(`docs/평가보고서.md`, 종합 4.4/5).

### ⑫ 평가보고서 개선사항 반영 (높음→중간→낮음)
- **높음**: 퀴즈 결과 서버 저장(`webpro_quiz_results` 테이블·RLS, 로그인 시 최고점 동기화/비로그인 localStorage 폴백), 표기 수치·용어 라벨 버그 정정.
- **중간**: 라우트 단위 `React.lazy`+Suspense 코드 분할(구문강조 230KB 등 메인 분리), `:focus-visible` 접근성, 모바일 메뉴 트랙 그룹화, web·ops 실습 랩 보강(27챕터 전부 68개).
- **낮음**: 라우트별 동적 메타(`usePageMeta`), **vitest 자동화 테스트 + CI 테스트 단계**, 번들 분석(`rollup-plugin-visualizer`, `npm run analyze`).

### ⑬ 코칭·가이드 메뉴 + Tips! 모음
- rest 리포 `learningData.ts`(coachingTopics)를 마크다운으로 충실히 이식.
- `/coaching` 3탭: **기술 코칭 4회차** / **부록 가이드 9종**(rest 7 + bootcamp 신규 2: SEO·OG 캐시, Google·Kakao 개발자 콘솔 설정) / **Tips! 모음 17개**(콜아웃 수집·검색·출처 링크).

## 3. 아키텍처

```
src/
  pages/      Home · Curriculum · Track · Lesson · Projects · ProjectDetail · Quiz · Resources · MyPage · Login · About · AuthCallback · NotFound
  components/ Header(병기 네비) · Footer · Logo · Reveal · Markdown · CodeBlock · ui(ChapterCard 등)
  demos/      apps/*.jsx  (프론트 라이브 데모 = 표시 소스)
  hooks/      useAuth · useProgress · useReveal
  lib/        supabase · db
  data/       curriculum.js · projects.js · projectCode.js · labs.js · review.js · materials(삭제됨)
  content/    <chapterId>.md  (27개 챕터 본문, lazy 로드)
  theme.js · styles/global.css
scripts/gen-og.mjs · supabase/schema.sql · .github/workflows/deploy.yml
docs/  (커리큘럼 JSON · 개발일지 · 평가보고서)
```

핵심 설계
- **트랙 시스템**: `TRACKS`(web/react/ai/ops) + `theme.js`의 trackColor/Grad/Hero/Light 헬퍼로 색상 일관 적용.
- **콘텐츠 분리**: 본문은 `src/content/*.md`, 메타데이터는 `curriculum.js`, 용어·퀴즈는 `review.js`로 분리.
- **lazy 로딩**: `import.meta.glob('./*.md', {query:'?raw'})`로 챕터 본문을 청크 분할.
- **진도**: 섹션 단위 낙관적 업데이트 → `webpro_progress` upsert(onConflict user_id,chapter_id,section_no).

## 4. 배포

- GitHub Actions: `main` 푸시 → `npm ci` → **`npm test`**(데이터 무결성, 실패 시 중단) → build(Secrets 주입) → `dist/404.html`(SPA fallback) → Pages.
- 커스텀 도메인 `public/CNAME`, vite `base:'/'`.
- OG 이미지는 한글 폰트가 있는 로컬에서 생성·커밋(CI 재생성 제거).
