## 1. 개요

### 1.1 플랫폼 소개
DreamIT 부트캠프는 **웹 기초(HTML·CSS·JS) → React 프론트엔드 → 생성형 AI 웹 서비스(FastAPI·Hugging Face) → 실전 배포·협업(Git·Docker·AWS)** 까지를 하나의 SPA에서 학습하는 한국어 실무형 웹 개발 학습 플랫폼이다. 교재 내용을 웹페이지로 재구성하고(원본 파일 비배포), 전문가 코칭·부록 가이드까지 갖춰 "개념 → 실습 → 프로젝트 → 검증 → 코칭"의 완결된 학습 루프를 제공한다.

### 1.2 규모 (실측치)
데이터 파일을 직접 로드해 집계한 수치.

| 항목 | 수치 | 근거 |
|------|------|------|
| 트랙 | **4** (web / react / ai / ops) | `TRACKS` |
| 챕터 | **27** (web 5 · react 8 · ai 8 · ops 6) | `TOTAL_CHAPTERS` |
| 강의(섹션) | **137** | `TOTAL_LESSONS` |
| 본문 마크다운 | 40개 파일(챕터 27 + 가이드 13), 총 **29,000+줄** | `src/content/` |
| 실습 랩 | **68** (27개 챕터 전부) | `labs.js` |
| 프로젝트 | **18** (입문·중급·고급 각 6, 상세=구현코드+동작 데모) | `projects.js` |
| 핵심 용어 | **216** | `TOTAL_TERMS` |
| 퀴즈 | **135문항** (27챕터 전부 커버) | `TOTAL_QUIZ` |
| 코칭·부록 가이드 | **13** (기술코칭 4 + 부록 9) | `coaching.js` |
| 실전 Tips! | **17** | `tips.js` |
| 자동화 테스트 | **29** (데이터 무결성 15 + 페이지 렌더 스모크 14) | vitest |

콘텐츠 커버리지가 균일하다. 27개 챕터 전부가 본문·퀴즈·랩을 갖춰 "데이터만 있고 본문은 빈" 미완성 패턴이 없다.

### 1.3 기술 스택
React 18 · Vite 5 · react-router-dom 6 · Supabase(Auth/Postgres/RLS) · react-markdown + react-syntax-highlighter(구문강조·녹색 주석·복사) · sharp(OG) · vitest + @testing-library/react(테스트) · rollup-plugin-visualizer(번들 분석).

### 1.4 배포 방식
GitHub Actions: `main` 푸시 → `npm ci` → **`npm test`(데이터·렌더 검증, 실패 시 중단)** → build(Secrets 주입) → `dist/404.html`(SPA fallback) → GitHub Pages. 커스텀 도메인(`bootcamp.dreamitbiz.com`, Cloudflare 경유), `base:'/'`. OG 이미지는 한글 폰트 있는 로컬에서 생성·커밋(`og-image-v2.png`).

## 2. 콘텐츠 평가  ★★★★★ (5.0/5)

### 2.1 4개 트랙 구성과 깊이
입문(웹 기초)부터 실전 배포까지 난이도 사다리가 끊김 없이 이어진다. 각 챕터는 설명 프로즈 + 표 + 실행 가능한 코드 + 핵심 포인트로 구성되며, 좌측 목차(섹션 진도 체크)와 우측 본문으로 렌더된다.

### 2.2 코드/주석 품질
모든 코드블록이 구문 강조되고 **주석이 녹색으로 강조**되며 복사 버튼을 제공한다. 백엔드/AI 챕터는 빈 줄·구분선·docstring을 갖춘 실행 가능한 형태다.

### 2.3 실습 예제 & 프로젝트
27개 챕터 전부 실습 랩(68개)을 갖췄고, 18개 프로젝트는 상세 페이지에서 **실제 구현 코드 + 동작 확인**(프론트 6종은 라이브 데모, 데모=표시 소스 일치 / 백엔드·배포는 결과 미리보기)을 제공한다.

### 2.4 학습 흐름
**개념(강의) → 실습(랩·프로젝트) → 검증(용어·퀴즈) → 코칭(가이드·Tips)** 의 닫힌 루프. 메인 EXPLORE 섹션과 마이페이지가 각 단계로 안내한다.

## 3. 기능 평가  ★★★★★ (4.7/5)

### 3.1 인증 (Google / Kakao)
Supabase OAuth, 로그인 시 프로필 upsert, `/auth/callback` 처리. 운영 콘솔에서 Google·Kakao·Email Provider 활성화 확인.

### 3.2 진도 저장 (Supabase RLS)
섹션 단위 낙관적 업데이트 → `webpro_progress` upsert. 전 테이블 RLS 본인 한정. 스키마 실행 완료 확인.

### 3.3 퀴즈·용어 (개선됨)
용어 216개(검색·트랙 필터·빈 상태 처리) + 퀴즈 135문항(챕터별/랜덤, 채점·해설). **최고점은 로그인 시 `webpro_quiz_results`(RLS)에 서버 저장**되어 기기 간 동기화되고, 비로그인은 localStorage 폴백. **마이페이지에 퀴즈 평균 최고점·챕터별 점수 배지로 노출.**

### 3.4 프로젝트 라이브 데모
프론트 프로젝트는 페이지 안에서 실제로 동작(Todo·포모도로·마크다운·칸반 등), 소스는 `?raw`로 데모와 동일 파일을 표시.

### 3.5 자료실 & 코칭·가이드 (신규)
자료실(전 챕터 목차·실습 모아보기) 외에 **코칭 메뉴(`/coaching`)** 신설 — 기술 코칭 4회차 + 부록 가이드 9종(SEO·OG, OAuth 콘솔 설정 등 운영 노하우 포함) + **Tips! 모음 17개**.

### 3.6 OG / SEO (개선됨)
OG 풀세트 + 카드. CI 한글 폰트 부재로 깨지던 이슈를 로컬 생성·커밋으로 해결, 캐시 무력화 위해 `og-image-v2.png` 버전. **라우트별 동적 `document.title`/description**(`usePageMeta`)을 전 주요 페이지에 적용, 404는 `noindex`.

> 한계: 정적 OG는 단일 이미지(소셜 크롤러는 JS 미실행). 라우트별 OG 이미지는 SSR/프리렌더가 필요.

## 4. UX·디자인 평가  ★★★★☆ (4.5/5)

### 4.1 디자인 시스템
NOVAWORKS 기반 일관된 토큰(트랙별 색상 web=violet·react=blue·ai=orange·ops=green), 다크/라이트 섹션, 플로팅 필 헤더.

### 4.2 반응형 & 접근성
`clamp()` 기반 유동 레이아웃, 모바일 목차·진도 체크 유지. **`:focus-visible` 포커스 링**과 **`prefers-reduced-motion` 시 애니메이션 비활성화** 적용. 검색 결과 빈 상태 처리.

### 4.3 네비게이션
영문+한글 병기, 활성=밑줄, 모바일은 학습 트랙 그룹화. 상단 메뉴 10개는 `≤1400px`에서 버거로 전환.

### 4.4 마이페이지 대시보드 (개선됨)
전체 진도율 + **퀴즈 평균 최고점** 2카드, 트랙·챕터별 진도와 퀴즈 점수 배지, 코칭 바로가기 — 저장되는 학습 활동을 모두 반영.

## 5. 기술 평가  ★★★★★ (4.6/5)

### 5.1 구조 / 코드 분할
페이지·데이터·콘텐츠 분리. **라우트 단위 `React.lazy`+Suspense**로 구문강조(230KB) 등을 메인 번들에서 분리. 챕터 본문은 `import.meta.glob ?raw`로 챕터별 lazy 청크.

### 5.2 마크다운 렌더링
react-markdown + remark-gfm + 커스텀 CodeBlock(구문강조·복사·녹색 주석). 표·콜아웃·헤딩 앵커 처리.

### 5.3 자동화 테스트 (신규)
vitest로 **데이터 무결성(15) + 페이지 렌더 스모크(14, jsdom)** 총 29개. 렌더 스모크는 빌드가 못 잡는 런타임 오류(예: 데이터 모듈 참조 오류)를 마운트 단계에서 차단하며 CI에서 매 배포 실행.

### 5.4 CI/CD & 번들 모니터링
GitHub Actions 자동 배포(테스트 게이트 포함), `npm run analyze`(ANALYZE=1)로 `dist/stats.html` 번들 트리맵 생성.

### 5.5 잠재 리스크
- 메인 청크 ~472KB(gzip 153KB) — supabase-js + react가 대부분, 추가 분할 여지는 제한적.
- 정적 OG 단일화(SSR 부재).
- 미사용 `webpro_enrollments` 테이블/`T.enrollments` 키 잔존(무해).

## 6. 강점 (Top 7)
1. **완결성** — 27챕터 전부 본문·코드·랩·퀴즈를 갖춘 빈틈 없는 커버리지.
2. **닫힌 학습 루프** — 개념→실습→프로젝트→검증→코칭까지 한 사이트에서.
3. **실증형 프로젝트** — 프론트는 라이브 데모(소스=데모 일치), 백엔드는 결과 미리보기.
4. **운영 노하우의 내재화** — OG 캐시·OAuth 콘솔 설정 등 실제 운영 경험을 부록 가이드로.
5. **데이터·진도·퀴즈 영속화** — RLS 기반 서버 저장 + 마이페이지 대시보드.
6. **품질 가드레일** — 자동화 테스트(데이터+렌더) + CI 게이트 + 번들 분석.
7. **접근성·SEO 기본기** — focus-visible·reduced-motion·동적 메타·noindex.

## 7. 개선 권장사항 (반영 현황)

- ✅ 퀴즈 결과 서버 저장 · 표기 수치/용어 라벨 정정 · 코드 분할 · 접근성(focus-visible) · 자동화 테스트 · 동적 메타 · 번들 분석 · 모바일 메뉴 그룹화 · web/ops 실습 랩 보강.

### 7.2 2차 재평가 권장 — 반영 완료 ✅
- ✅ 마이페이지에 퀴즈 점수·코칭 노출 · Home/MyPage/Login/NotFound 메타(+404 noindex) · 검색 빈 상태 · `prefers-reduced-motion` · Login 친화 에러 · 미사용 `attempts` 제거 · 렌더 테스트 확장(Lesson/ProjectDetail/GuideDetail/MyPage).

### 7.3 남은 과제 (낮음)
- **데스크톱 트랙 드롭다운**: 상단 10개 항목을 "트랙▾"으로 묶으면 ≤1400px 외 폭에서도 여유. (현재 ≤1400px는 버거라 실사용 영향 작음.)
- **정적 OG 라우트화**: 프리렌더(예: vite-plugin-ssg)로 챕터·트랙별 공유 이미지/메타.
- **E2E 테스트**: 로그인·진도 토글·퀴즈 채점의 실제 브라우저 시나리오(Playwright).
- **미사용 `enrollments` 정리** 또는 "수강신청" 기능으로 활성화.

### 총평
1차 평가(4.4) 이후의 권장사항을 **빠짐없이 반영**하고, 코칭·가이드 메뉴와 마이페이지 대시보드로 "저장되지만 보이지 않던" 학습 활동까지 노출했다. 자동화 테스트(데이터+렌더)와 CI 게이트로 런타임 회귀를 구조적으로 차단해, 콘텐츠 완성도뿐 아니라 **운영 안정성과 유지보수성**까지 갖춘 교육 상용 플랫폼 수준에 도달했다. 남은 과제는 SSR 기반 라우트별 OG·E2E 테스트 같은 "한 단계 더"에 해당하는 고급 항목이다. **종합 4.7/5.0.**
