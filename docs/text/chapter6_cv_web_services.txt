6.1.1 이미지 분류란? (ResNet, ViT, ConvNeXt 등 모델 구조 개요)
이미지 분류 모델의 역사는 CN N (Convolutional N eural N etwork)에서 시작합니다. 2012년

AlexN et이 ImageN et 대회에서 압도적인 성능을 보인 이후, 더 깊고 정확한 CN N 모델들이

개발되었습니다. 2020년대에는 N LP에서 성공한 Transformer 구조가 이미지 분야로 이전되

어 ViT(Vision Transformer) 계열이 CN N 을 능가하기 시작했습니다.

이미지 분류의 핵심 처리 파이프라인을 단계별로 살펴봅니다.

   1. 이미지 로드: PIL 또는 O penCV로 이미지 읽기

   2. 전처리: 크기 조정(224× 224 등), 정규화 (채널 평균/표준편차 기준)

   3. 텐서 변환: numpy 배열 → PyTorch Tensor, [H ,W ,C] → [B,C,H ,W ]

   4. 모델 추론: model(tensor) → 클래스 점수 벡터 (logits)

   5. 후처리: Softmax → 확률값 → Top-K 클래스 추출

   6. 결과 반환: 클래스 레이블, 신뢰도 점수 JSO N 반환

# services/classification_service.py

import io, time, torch, base64

from typing import List, Dict, Optional

from PIL import Image

from transformers import pipeline



class ImageClassificationService:

   SUPPORTED_MODELS = {

       'vit-base':    'google/vit-base-patch16-224',

       'convnext-base': 'facebook/convnext-base-224-22k-1k',

       'resnet-50':    'microsoft/resnet-50',

       'efficientnet': 'google/efficientnet-b4',

       'deit-base':    'facebook/deit-base-distilled-patch16-224',

   }



   def __init__(self, model_key: str = 'vit-base'):

       self.model_key = model_key

       self.pipe = None

       self.device = 0 if torch.cuda.is_available() else -1



   def load_model(self, model_key: Optional[str] = None):

       key = model_key or self.model_key

       model_id = self.SUPPORTED_MODELS.get(key, self.SUPPORTED_MODELS['vit-base'])

       print(f'[Classification] Loading: {model_id}')

       self.pipe = pipeline(

           task='image-classification',

           model=model_id,

           device=self.device,

       )



                                                        — 7—
HuggingFace를 활용한 인공지능 웹 애플리케이션 개발 제6장 컴퓨터 비전 웹 서비스



      self.model_key = key



   def classify_from_file(self, image_bytes: bytes, top_k: int = 5) -> Dict:

      if self.pipe is None:

          raise RuntimeError('분류 모델이 로드되지 않았습니다.')

      start = time.time()

      image = Image.open(io.BytesIO(image_bytes)).convert('RGB')

      results = self.pipe(image, top_k=top_k)

      elapsed = round(time.time() - start, 3)

      return {

          'predictions': [

               {'rank': i+1, 'label': r['label'],

               'score': round(r['score'],6),

               'percentage': round(r['score']*100,2)}

               for i, r in enumerate(results)

          ],

          'top_prediction': results[0]['label'],

          'top_score': round(results[0]['score'],6),

          'model_used': self.SUPPORTED_MODELS[self.model_key],

          'inference_time_sec': elapsed,

          'image_size': f'{image.width}x{image.height}',

      }



   def classify_from_base64(self, b64_string: str, top_k: int = 5) -> Dict:

      if ',' in b64_string:

          b64_string = b64_string.split(',', 1)[1]

      return self.classify_from_file(base64.b64decode(b64_string), top_k)



   def classify_from_url(self, url: str, top_k: int = 5) -> Dict:

      import httpx

      resp = httpx.get(url, timeout=10)

      resp.raise_for_status()

      return self.classify_from_file(resp.content, top_k)

6.2 객체 탐지 API 만들기 (YO LO v8)
객체 탐지(O bject Detection)는 이미지 내에서 여러 객체의 위치(Bounding Box)와 클래스

(종류)를 동시에 예측하는 태스크입니다. 단순히 '이미지에 무엇이 있는가'를 넘어 '어디에 있

는가'까지 알려주는 더 복잡한 비전 태스크입니다.




6.2.1 객체 탐지란? 이미지 속 객체 위치(Box) + 클래스 예측
객체 탐지 결과는 탐지된 각 객체에 대해 바운딩 박스 좌표 [x1, y1, x2, y2], 클래스 이름, 신

뢰도 점수를 함께 반환합니다. IoU(Intersection over Union)와 N M S(N on-M aximum

Suppression)가 핵심 알고리즘입니다.
