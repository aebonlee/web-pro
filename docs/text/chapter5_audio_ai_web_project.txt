니다. 스마트 스피커, 음성 비서, 자동 자막, 오디오북 등 실생활 속 AI 음성 기술은 이미 수십

5.1 W hisper로 음성 인식 API
O penAI가 2022년 공개한 W hisper는 현재 세계 최고 수준의 다국어 음성 인식 모델입니다.

영어, 한국어를 포함한 99개 언어를 지원하며, H ugging Face H ub에도 공개되어 있어

Pipeline API 또는 openai-whisper 패키지로 바로 사용할 수 있습니다.




5.1.1 Hugging Face openai/whisper 모델 소개
W hisper는 680,000시간의 다국어 오디오 데이터로 학습된 Transformer 기반의 Encoder-

Decoder 모델입니다. 음성 신호를 먼저 80채널 Log-M el Spectrogram으로 변환한 후 인코더

에 입력하고, 디코더가 자동 회귀 방식으로 텍스트를 생성합니다.



모델 크기              파라미터                   디스크 크기               특징

tiny               39M                    ~ 75M B              가장 빠름. 실험/체험용.

base               74M                    ~ 145M B             속도와 정확도 균형. 학습/테스트 추천.

small              244M                   ~ 480M B             일반 사용에 충분한 품질.

medium             769M                   ~ 1.5G B             고품질. G PU 권장.

large-v3           1550M                  ~ 3.1G B             최고 정확도. A100 G PU 권장.




 팁: 작은 버전으로도 충분히 체험 가능
CPU 환경에서는 tiny 또는 base 모델을 권장합니다.

tiny: 약 10초 길이 오디오를 CPU에서 15~ 30초 내 처리 가능

base: CPU에서 30~ 60초 처리 시간. 한국어 인식도 양호.



G PU(T4) 환경에서는 medium 이상을 사용하면 실시간(1초 오디오 < 1초 처리) 가능.



H ugging Face Inference API를 사용하면 로컬 G PU 없이도 large-v3 이용 가능:
   from huggingface_hub import InferenceClient
   client = InferenceClient(token='hf_...')
   result = client.automatic_speech_recognition('audio.wav')

5.1.2 W hisper의 장점: 언어 감지 + 정확한 자막 생성
W hisper가 다른 ASR 모델 대비 특별히 강점을 갖는 영역은 다음과 같습니다.



W hisper의 강점                구체적인 내용

                            입력 오디오 첫 30초를 분석하여 99개 언어 중 자동 감지. 별도 언어 지
자동 언어 감지
                            정 불필요.

                            단어/문장 단위 타임스탬프 생성 가능. 자막(SRT/VTT) 자동 생성에 활
타임스탬프 지원
                            용.

                            배경 음악, 잡음이 있는 환경에서도 높은 인식률. 6dB 이상 SN R에서
잡음 환경 강인성
                            90%+ 정확도.

다국어 번역                      음성을 인식하면서 동시에 영어로 번역 가능 (translate 태스크).

                            30초 단위로 청크(Chunk) 분할 후 연속 처리. 수 시간 분량 오디오도
긴 오디오 처리
                            처리 가능.

구어체 강인성                     말끝 흐림, 겹쳐 말하기, 방언 등 자연스러운 대화 인식에 강함.



W hisper가 내부적으로 오디오를 처리하는 과정을 단계별로 살펴봅니다.

   1. 오디오 전처리: 입력 오디오를 16kH z 모노(M ono) WAV로 변환

   2. Log-M el Spectrogram 변환: 80채널, 30초 단위 스펙트로그램 생성 (3,000 프레임 ×
       80채널)

   3. 인코더(Encoder): Transformer 인코더로 오디오 특징(Feature) 추출

   4. 언어 감지: 디코더 첫 토큰 예측으로 언어 자동 판별

   5. 디코더(Decoder): 자기 회귀 방식으로 텍스트 토큰 순서대로 예측

   6. 타임스탬프 정렬: 각 단어의 시작/종료 시간 계산 (DTW 알고리즘)

class ASRService:

       """Whisper 파이프라인 로드"""

       model_id = self.SUPPORTED_MODELS.get(

           self.model_size, 'openai/whisper-base'

       )

 def transcribe_bytes(

    self,

    audio_bytes: bytes,

    suffix: str = '.wav',

    **kwargs

 ) -> Dict[str, Any]:

    """바이트 데이터로 직접 변환 (임시 파일 사용)"""

    with tempfile.NamedTemporaryFile(suffix=suffix, delete=False) as f:

        f.write(audio_bytes)

        tmp_path = f.name

    try:

        return self.transcribe(tmp_path, **kwargs)

    finally:

        os.unlink(tmp_path) # 임시 파일 삭제



 def generate_srt(

    self,

    audio_path: str,

    language: Optional[str] = None,

 ) -> str:

    """SRT 형식 자막 파일 생성"""

    result = self.transcribe(

        audio_path,

        language=language,

        return_timestamps=True,

    )

    chunks = result.get('chunks', [])

    if not chunks:

        return f'1\n00:00:00,000 --> 00:10:00,000\n{result["text"]}\n'

HuggingFace를 활용한 인공지능 웹 애플리케이션 개발 제5장 오디오 인공지능 웹 프로젝트




      srt_lines = []

      for i, chunk in enumerate(chunks, 1):

         ts = chunk.get('timestamp', (0, 1))

         start = self._sec_to_srt_time(ts[0] or 0)

         end    = self._sec_to_srt_time(ts[1] or (ts[0] or 0) + 2)

         srt_lines.append(f'{i}')

         srt_lines.append(f'{start} --> {end}')

         srt_lines.append(chunk['text'].strip())

         srt_lines.append('')

      return '\n'.join(srt_lines)



   @staticmethod

   def _sec_to_srt_time(seconds: float) -> str:

      """초 → SRT 타임코드 변환 (hh:mm:ss,mmm)"""

      ms = int((seconds % 1) * 1000)

      s = int(seconds) % 60

      m = (int(seconds) // 60) % 60

      h = int(seconds) // 3600

      return f'{h:02d}:{m:02d}:{s:02d},{ms:03d}'

def convert_to_wav(

   input_path: str,

   output_path: Optional[str] = None,

   sample_rate: int = 16000,

   channels: int = 1,

) -> str:

   """

   오디오 파일을 Whisper 입력 형식(16kHz, 모노 WAV)으로 변환합니다.

   """

   if output_path is None:

         output_path = str(TEMP_DIR / f'{uuid.uuid4()}.wav')



   if PYDUB_AVAILABLE:

         audio = AudioSegment.from_file(input_path)

         audio = audio.set_frame_rate(sample_rate)

         audio = audio.set_channels(channels)

         audio.export(output_path, format='wav')

   else:

         # soundfile로 읽고 scipy로 리샘플링

         import scipy.signal as signal

         data, orig_sr = sf.read(input_path)

         if orig_sr != sample_rate:

             num_samples = int(len(data) * sample_rate / orig_sr)

             data = signal.resample(data, num_samples)

         if data.ndim > 1:

             data = data.mean(axis=1) # 스테레오 → 모노

         sf.write(output_path, data, sample_rate)

   return output_path

def get_audio_info(audio_path: str) -> dict:

   """오디오 파일 기본 정보 반환"""

   try:

      data, sr = sf.read(audio_path)

      duration = len(data) / sr

      channels = 1 if data.ndim == 1 else data.shape[1]

      return {

          'duration_sec': round(duration, 2),

          'sample_rate': sr,

          'channels': channels,

          'samples': len(data),

          'format': Path(audio_path).suffix.lower(),

      }

   except Exception as e:

      return {'error': str(e)}

5.2.1 Hugging Face suno/bark 또는 SpeechT5 모델 활용

M icrosoft의 T5 기반 다목적 음성 처리 모델로, 더 빠른 처리 속도와 안정성을 제공합니다.

            f.write(chunk)

            total_size += len(chunk)



     return {

         'total_size_mb': round(total_size / 1024 / 1024, 2)

FastAPI UploadFile 사용 시 꼭 알아야 할 핵심 사항:



1. 항상 await file.read()로 비동기 읽기

  - file.read() (동기) 사용 시 블로킹 발생



2. 대용량 파일은 청크 읽기 사용

  - while chunk := await file.read(1024*1024): ...



3. multipart/form-data 헤더 필수

  - Swagger UI는 자동 처리

  - curl: -F 'file=@path.wav' (Content-Type 자동 설정)

  - Python requests: files={'file': open('audio.wav', 'rb')}



4. 파일명 인코딩 주의

  - 한글 파일명은 urllib.parse.unquote(file.filename) 처리 권장



5. content_type 신뢰 금지

  - 클라이언트가 임의 설정 가능. 파일 확장자로 검증 권장.
