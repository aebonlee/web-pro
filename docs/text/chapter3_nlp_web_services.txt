 이 장의 개요
이 장에서는 H ugging Face의 다양한 자연어처리(N LP) 모델을 FastAPI와 연동하여 실제 웹 API

서비스 형태로 구현합니다. 단순한 챗봇부터 시작해서 요약기, 번역기, 문서 유사도 검색기까지

실습하며, 사용자 요청을 받아 모델이 예측을 수행하고 결과를 응답하는 AI 웹 서비스의

전체 흐름을 이해하게 됩니다. 또한 Swagger UI 기반으로 프론트엔드 없이 API를 체험하고

REST API 설계와 데이터 흐름에 집중합니다.




자연어 처리(N LP, N atural Language Processing)는 인공지능 분야에서 가장 활발히 응용되

고 있는 영역 중 하나입니다. 텍스트를 이해하고, 변환하고, 생성하는 N LP 기술은 챗봇, 번역,

검색, 요약 등 수많은 실용 서비스의 핵심 엔진으로 자리 잡았습니다.

H ugging Face의 Transformers 라이브러리는 BERT, G PT-2, T5, M arianM T, sentence-

transformers 등 수백 가지 N LP 모델을 단일 API로 제공하며, FastAPI와 결합하면 이러한 모

델들을 빠르게 웹 서비스로 배포할 수 있습니다.

이 장에서 구현할 서비스 목록은 다음과 같습니다.

3.1 FastAPI 챗봇 만들기 (단순 텍스트 입출력)
챗봇(Chatbot)은 사용자의 텍스트 입력을 받아 자연스러운 응답을 생성하는 가장 직관적인

N LP 서비스입니다. 이 절에서는 H ugging Face의 텍스트 생성 모델을 FastAPI로 래핑하여

REST API 형태의 챗봇을 만듭니다.




3.1.1 챗봇의 기본 원리: 입력 텍스트 → 모델 추론 → 응답 텍스트
챗봇의 핵심 동작 원리는 단순합니다. 사용자가 입력한 텍스트(prompt)를 언어 모델에게 전

달하면, 모델은 사전 학습된 지식을 기반으로 다음에 이어질 확률이 높은 토큰(token)들을 순

서대로 생성하여 응답 텍스트를 만들어냅니다.



단계                 설명                         코드 역할

① 사용자 입력           H TTP PO ST 요청으로 텍스트 전달    FastAPI 엔드포인트가 수신

② 토크나이징            텍스트 → 토큰 ID 시퀀스 변환         AutoTokenizer.encode()

③ 모델 추론            토큰 시퀀스 → 다음 토큰 예측          model.generate()

④ 디코딩              토큰 ID 시퀀스 → 텍스트 변환         tokenizer.decode()

⑤ 응답 반환            JSO N 형태로 클라이언트에 전달        FastAPI Response



텍스트 생성 모델의 동작 방식을 더 깊이 이해하려면 '생성 전략(Decoding Strategy)'에 대해

알아야 합니다. 어떤 전략을 선택하느냐에 따라 생성되는 텍스트의 품질과 다양성이 크게 달

라집니다.

3.1.3 대화 이력을 저장하지 않는 단순 챗팅 구조
이 절에서 구현하는 챗봇은 '무상태(Stateless)' 구조입니다. 즉, 각 API 요청은 이전 대화 이력

을 기억하지 않고 독립적으로 처리됩니다. 이는 서버 구현을 단순하게 유지하고, 수평 확장

(H orizontal Scaling)이 용이하다는 장점이 있습니다.



구조 유형                               특징

                                    각 요청이 독립적. 이전 대화 기억 없음. 서버 상태 관리 불필요. 확장 용
무상태 챗봇 (이 절 구현)
                                    이.

                                    대화 히스토리를 서버(Redis 등) 또는 클라이언트에 저장. 문맥 연속성 지
상태 유지 챗봇 (심화)
                                    원.

3.2 요약기 API 만들기 (T5 기반)
문서 요약(Text Summarization)은 긴 텍스트에서 핵심 내용을 추출하거나 재구성하여 짧은

형태로 만드는 N LP 태스크입니다. 이 절에서는 T5(Text-to-Text Transfer Transformer) 아

키텍처를 기반으로 한 요약 모델을 FastAPI로 구현합니다.




3.2.1 T5 모델 개념 소개 ('summarize: ' prefix 사용)
T5는 2019년 G oogle이 발표한 모델로, '모든 N LP 태스크를 텍스트→텍스트 변환 문제로 통

일'한다는 혁신적인 아이디어를 제시했습니다. 번역, 요약, 분류, 질의응답 등 다양한 태스크

를 별도의 출력 헤드 없이 단일 Seq2Seq 구조로 처리합니다.



T5의 핵심은 태스크 prefix입니다. 입력 텍스트 앞에 어떤 태스크를 수행할지를 나타내는 텍

스트 접두사를 붙여 모델에게 의도를 전달합니다.



T5 태스크 Prefix               예시 입력 형식

요약 (Summarize)              summarize: The stock market fell sharply today...

번역 (Translate)              translate English to G erman: H ello, how are you?

문법 교정 (G rammar)            grammar: She go to school yesterday.

질의응답 (Q & A)                question: W hat is the capital? context: Paris is...




 T5 주요 변형 모델
t5-small    : 60M 파라미터. 가볍고 빠름. 학습/테스트 환경에 적합.

t5-base     : 220M 파라미터. 성능과 속도의 균형.

t5-large    : 770M 파라미터. 높은 품질. G PU 권장.

google/flan-t5-base : instruction-tuned 버전. 더 다양한 태스크 지원.

KETI-AIR/ke-t5-base : 한국어 특화 T5. 한글 텍스트 요약에 최적화.

async def preview_text(text: str, language: Language = Language.korean):

   if language == Language.korean:

       tokenizer = summarizer_service.ko_tokenizer

   else:

       tokenizer = summarizer_service.en_tokenizer



   if tokenizer is None:

       raise HTTPException(status_code=503, detail='토크나이저 미로드')



   token_count = len(tokenizer.encode(text))

   is_truncatable = token_count > 512



   return {

       'preview': text[:200] + ('...' if len(text) > 200 else ''),

       'total_chars': len(text),

       'total_tokens': token_count,

       'will_be_truncated': is_truncatable,

       'max_supported_tokens': 512 if language == Language.korean else 1024,

   }

            results.append(doc)
