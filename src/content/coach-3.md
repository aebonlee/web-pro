# 3회차 · LLM 최적화 코칭

LLM API 호출 최적화와 프롬프트 튜닝을 코칭합니다.

## 코칭 목표

- 프롬프트 품질 개선
- API 호출 비용 최적화
- 응답 품질 향상 전략

## 주요 내용

LLM API의 비용과 성능을 최적화합니다. 프롬프트를 정교하게 다듬고, 캐싱 전략을 적용하며, 응답의 일관성을 높이는 방법을 학습합니다.

## 최적화 기법

- 시스템 프롬프트 정교화
- 온도(temperature)와 max_tokens 최적값
- 응답 캐싱으로 비용 절감
- 폴백(fallback) 모델 전략

## 프롬프트 4대 요소

- 역할(system) — "너는 ○○ 전문가다"로 톤·관점 고정
- 맥락 — 입력 데이터·상황을 구조화해 전달
- 제약 — 분량·언어·금지(과장 금지 등)를 명시
- 출력 형식 — JSON 스키마를 강제해 파싱을 안정화

## JSON 스키마 강제 (핵심 패턴)

```typescript
const out = await ask(
  '너는 창업 코치다. 반드시 JSON만: {"score":0,"tips":["..."]}',
  userInput,
  { json: true, temperature: 0.5, max_tokens: 600 }   // response_format: json_object
);
const data = JSON.parse(out);   // 형식이 고정돼 파싱 실패가 줄어든다
```

## 비용·속도 최적화

| 기법 | 효과 |
| --- | --- |
| JSON 출력 강제 | 파싱 안정 · 재요청 감소 |
| max_tokens 제한 | 비용·응답 속도 개선 |
| temperature 낮춤(0.3~0.6) | 일관된 결과 |
| 응답 캐싱 | 동일 입력 비용 0 |
| 폴백(키 없을 때) | 오프라인에서도 동작 · 시연 안전 |

> 💡 **Tip!** 구현 예시 26개가 전부 이 패턴입니다 — system 프롬프트로 JSON 스키마를 강제하고, 키가 없으면 휴리스틱 폴백으로 동작. "프로젝트 구현 예시" 갤러리의 파이프라인 탭에서 직접 확인하세요.
